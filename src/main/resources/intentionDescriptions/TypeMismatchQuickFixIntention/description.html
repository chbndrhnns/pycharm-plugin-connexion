Shows the full "expected type vs actual type" mismatch details in a balloon when the caret is on a corresponding editor highlight.

This intention becomes available when the caret is placed on a highlighted type mismatch (for example, "Expected type 'Int', got 'Any'"). When invoked, it reads the visible highlight at the caret and displays its full description in a balloon notification. No code changes are performed.

Implementation notes:

  • Uses only public APIs by scanning the document's markup model (DocumentMarkupModel + MarkupModelEx) for RangeHighlighters overlapping the caret.
  • Selects the best matching highlighter (prefers higher layer; tie-breaker by smaller covered range) and extracts the tooltip text.
  • Heuristically detects messages containing both "expected â€¦ type" and one of "actual"/"got"/"found" (case-insensitive).

Example: 


fun example(x: Any) {
    val y: Int = x // Expected type 'Int', got 'Any'
}

Tip: Place the caret on the underlined mismatch and press Alt+Enter to invoke "Show type mismatch details".
