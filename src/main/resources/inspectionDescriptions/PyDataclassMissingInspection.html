Reports classes that inherit from a @dataclass but are not decorated with @dataclass themselves.

When a class inherits from a dataclass, the dataclass machinery is not automatically applied to the subclass. If the subclass defines new fields or requires dataclass behavior, it should also be decorated with @dataclass.

Example:


import dataclasses

@dataclasses.dataclass
class Base:
    x: int

# Warning: Subclass of a dataclass should also be decorated with @dataclass
class Derived(Base):
    y: int
