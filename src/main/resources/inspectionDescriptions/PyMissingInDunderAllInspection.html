Symbol not exported in __all__

Reports public top-level symbols in a Python module that are not exported via __all__.

  • If a module defines __all__, all public classes, functions, and variables defined at the top level should be listed there.
  • If a private module (file name starts with an underscore) lives inside a package that defines __all__, its public symbols should be re-exported via the package's __all__.
  • A quick-fix is available for the module-level case to append the missing symbol name to the module's __all__ list or tuple.
